17  /** We provide a simple set of classes for accessing core HDF5 functionality
18  from C++. This was needed since
19  - The HDF5 C interface is not trivial to use peroperly, particularly in
20  regards to data type conversions and resource management
21  - It is very easy to use the C interface incorrectly without knowning it.
22  - The \external{http://hdf5.org, standard HDF5 C++ interface} doesn't
23  really simplify use of the HDF5 library and doesn't make use of the
24  features of C++.
25 
26  The main classes provide access too:
27  - HDF5 files through RMF::File and RMF::HDF5::ConstFile
28  - HDF5 groups through RMF::Group and RMF::HDF5::ConstGroup
29  - HDF5 data sets through RMF::DataSetD and RMF::HDF5::ConstDataSetD
30 
31  The \c Const variants are for read only files.
32 
33  They allow one to create a manipulate data sets containing floating point
34  values, integers, strings, variable length arrays of floating point values
35  and a few other types. Attributes on data sets of groups of those types
36  can also be manipulated.
37 
38  The top level classes are, in turn, derived from RMF::Object,
39  RMF::HDF5::ConstAttributes and RMF::HDF5::MutableAttributes which provide
40  access to general HDF5 per-object and attribute functionality (all objects
41  in HDF5 can have arbitrary small pieces of data attached to them via
42  attributes).
43 
44  Not all of the functionality of the HDF5 \c C library is covered in C++.
45  You can get the corresponding native handle from most objects with methods
46  like RMF::File::get_handle() and use that for other operations if needed.
47  Or submit a patch to the library.
48  */
