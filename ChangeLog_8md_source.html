1 # Change Log #
2 - RMF now uses the new avro-based format with the `.rmf` suffix.
3 - RMF::decorator::Bond and RMF::decorator::BondConst now return
4  RMF::NodeHandle and RMF::NodeConstHandle instead of ints.
5 - RMF::decorator::Representation and RMF::decorator::RepresentationConst now return
6  RMF::NodeHandles and RMF::NodeConstHandles instead of ints.
7 - RMF::NodeConstHandle::get_index() went away as it was a duplicate of
8  RMF::NodeConstHandle::get_id()
9 - RMF::NodeType and RMF::FrameType were replaced with RMF::Enum
10  instances. This increases type safety in python and makes
11  converting to and from strings easier (you can now use `str()` in python).
12 - RMF::decorator::Residue::get_residue_index() and
13  RMF::decorator::Residue::get_residue_type() replace the versions
14  without `residue` in the name
15 - Decorators and factories have been moved into the RMF::decorator namespace in C++.
16 - Two new formats have been added: `.rmf3` and its compressed version
17  `.rmfz`. You should use those in preference to the prior formats
18  for new files.
19 - The RMF repository now has a git submodule. This means you need to do
20  `git submodule init` after cloning (or `git clone
21  git@github.com:salilab/RMF.git --recursive`) and then update with
22  `git pull --recursive origin develop`.
23 - You can now only either create a new RMF or open an existing one read
24  only. Most backends only supported those two anyway.
25 - Managing of data buffers has been revamped and wrapped in the
26  RMF::BufferHandle class which tracks a shared buffer.
27 - Indexes and NodeID attributes are now just ints.
28 - You now get keys by passing the appropriate traits object instead of
29  having the type in the function name: eg `f.get_key(cat, "name", RMF.IntTraits())`
30 - Appropriate decorators now return RMF::Vector3, RMF::Vector4 or
31  RMF::Vector3s instead of RMF::Floats or RMF::FloatsList.
32 - There are now static and non-static versions of the decorators.
33 - You now have to explicitly add a frame or set the current frame before
34  manipulating frame data.
35 - RMF::NodeConstHandle::get_value() subsumes get_value_always() which was
36  removed. If there is no value, then either a RMF::Nullable for
37  which RMF::Nullable::get_is_null() is true is returned (C++) or
38  `None` is returned (python).
39 - RMF::FileHandle::get_node_from_id() was renamed to
40  RMF::FileHandle::get_node()
41 - Various methods that took ints to identify frames now expect RMF::FrameID.
42 - A VMD plugin was added
