3 # Mapping data to RMF files # {#mapping}
4 
5 This page outlines how one would go about storing various types of data in RMF files. Much of what it says is merely suggestions, and is not mandatory.
6 
7 For nodes types, the following node shapes are used
8 \dot
9  digraph key {
10  root [label="RMF::ROOT" shape=circle URL="\ref RMF::ROOT"];
11  rep [label="RMF::REPRESENTATION" shape=oval URL="\ref RMF::REPRESENTATION"];
12  bond [label="RMF::BOND" shape=invhouse URL="\ref RMF::BOND"];
13  org [label="RMF::ORGANIZATIONAL" shape=folder URL="\ref RMF::ORGANIZATIONAL"];
14  feat [label="RMF::FEATURE" shape=hexagon URL="\ref RMF::FEATURE"];
15  tree [label="elided subtree" shape=triangle];
16  }
17 \enddot
18 And a dot means an RMF::ALIAS node. The key concept to keep in mind with RMF::ALIAS nodes is that, unlike everything else, they don't introduce a new entity, simply point to one that exists elsewhere.
19 See RMF::ROOT, RMF::REPRESENTATION, RMF::BOND, RMF::ORGANIZATIONAL, RMF::FEATURE for more information.

21 # PDB files #
22 
23 PDB files are most naturally mapped as follows:
24 - each PDB chain is a node that is a child of the root and stores data using the RMF::decorator::Chain decorator
25 - for a protein, each residue is a node that is the child of the respective chain and stores data using the RMF::decorator::Residue decorator
26 - each atom is a node that is the child of its residue or molecule. It stores data using the RMF::decorator::Atom and RMF::decorator::Particle decorators
27 
28 All bonds are stored as RMF::BOND nodes with two children for the endpoints of the bond. The bonds are stored as children of the first common ancestor of the endpoints.
29 
30 \dot
31  digraph pdb {
32  root [label="root" shape=circle];
33  chain [label="Chain A" shape=folder];
34  r0 [label="Residue 0 (RMF::decorator::Residue)" shape=folder URL="\ref RMF::decorator::Residue"];
35  r1 [label="Residue 1 (RMF::decorator::Residue)" shape=folder URL="\ref RMF::decorator::Residue"];
36  br [label="bond" shape=invhouse];
37  br0 [label="bonded" shape=point];
38  br1 [label="bonded" shape=point];
39  rdot [label="Residue ... (RMF::decorator::Residue)" shape=folder URL="\ref RMF::decorator::Residue"];
40  a0 [label="N (RMF::decorator::Atom, RMF::decorator::Particle)" shape=oval URL="\ref RMF::decorator::Atom"];
41  a1 [label="CA (RMF::decorator::Atom, RMF::decorator::Particle)" shape=oval URL="\ref RMF::decorator::Atom"];
42  a2 [label="C (RMF::decorator::Atom, RMF::decorator::Particle)" shape=oval URL="\ref RMF::decorator::Atom"];
43  ap0 [label="N (RMF::decorator::Atom, RMF::decorator::Particle)" shape=oval URL="\ref RMF::decorator::Atom"];
44  apdot [label="... (RMF::decorator::Atom, RMF::decorator::Particle)" shape=oval URL="\ref RMF::decorator::Atom"];
45  b0 [label="bond" shape=invhouse];
46  b00 [label="bonded" shape=point];
47  b01 [label="bonded" shape=point];
48  b1 [label="bond" shape=invhouse];
49  b10 [label="bonded" shape=point];

52  chain->r0;
53  chain->br;
54  chain->r1;
55  chain->rdot;

57  r0->a1;
58  r0->a2;

78 If the PDB file has more than one model, all with the same structure, the models should be stored as frames in the RMF file.

80 # Rigid fits of proteins into EM maps or other data #
81 
82 When you have a number of structures that are formed by applying a rigid transform to a set of base structures (eg fitting pdb structures into an EM map), they can be stored as follows:
83 - RMF::decorator::ReferenceFrame nodes can be used to store the rigid transforms
84 - under each RMF::decorator::ReferenceFrame node, you can store one of
85  - a RMF::decorator::External referencing an external structure
86  - an RMF hierarchy detailing the structure being transformed
87 Multiple RMF::decorator::External or the child hierarchies can refer to the same structure. Hopefully software loading the RMF file will take advantage of that.
88 
89 Multiple structures can be stored as separate frames.
90 
91 \dot
92  digraph rigid {
93  root [label="root" shape=circle];
94  copy_0 [label="Copy 0 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
95  copy_1 [label="Copy 1 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
96  copy_2 [label="Copy 2 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
97  pdb [label="External PDB (RMF::decorator::External)" shape=oval URL="\ref RMF::decorator::External"];
98  em [label="External MRC (RMF::decorator::External)" shape=oval URL="\ref RMF::decorator::External"];
99  root -> copy_0;
100  root -> copy_1;
101  root -> copy_2;
102  root->em;
103  copy_0 -> pdb;
104  copy_1 -> pdb;
105  copy_2 -> pdb;
106  }
107 \enddot

109 # Structures with multiple organization schemes #
110 
111 If the file contains data that has multiple organization schemes, for example, grouping by type of protein, which unit cell etc, several different hierarchies can be provided, all with the same child structures using RMF::ALIAS nodes to provide the different organization schemes.
112 
113 \dot
114  digraph symmetry {
115  root [label="root" shape=circle];
116  unit_cell [label="structure" shape=folder];
117  a [label="Protein A" shape=triangle];
118  b [label="Protein B" shape=triangle];
119  c [label="Protein C" shape=triangle];
120  d [label="Protein D" shape=triangle];
121  bt [label="By fold" shape=folder];
122  bth [label="Beta barrel" shape=folder];
123  btx [label="Alpha helical" shape=folder];
124  bta [label="Protein A" shape=point];
125  btb [label="Protein B" shape=point];
126  btc [label="Protein C" shape=point];
127  btd [label="Protein D" shape=point];
128  root->unit_cell;
129  unit_cell->a;
130  unit_cell->b;
131  unit_cell->c;
132  unit_cell->d;
133  root->bt;
134  bt->btx;
135  bt->bth;
136  btx->bta;
137  btx->btb;
138  bth->btc;
139  bth->btd;
140  bta->a;
141  btb->b;
142  btc->c;
143  btd->d;
144  }
145 \enddot

147 # Symmetries #
148 
149 When there is symmetry, or other repeated unit, define the unit cell in one sub hierarchy and then add RMF::decorator::ReferenceFrame nodes that have that unit cell as a child to define the symmetric parts.
150 
151 \dot
152  digraph symmetry {
153  root [label="root" shape=circle];
154  unit_cell [label="unit cell" shape=folder];
155  structure [label="structural nodes" shape=triangle];
156  copy_0 [label="Copy 0 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
157  copy_1 [label="Copy 1 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
158  copy_2 [label="Copy 2 (RMF::decorator::ReferenceFrame)" shape=folder URL="\ref RMF::decorator::ReferenceFrame"];
159  root -> unit_cell;
160  unit_cell -> structure
161  root -> copy_0;
162  copy_0 -> structure;
163  root -> copy_1;
164  copy_1 -> structure;
165  root -> copy_2;
166  copy_2 -> structure;
167  }
168 \enddot
