6 Once you build RMF, there will be a file called `rmfplugin.so` in the
7 `lib` directory. Creating a link to that in the
8 `vmd/plugins/YOUR_ARCHITECTURE/molfile/` directory, should make `vmd`
9 load the plugin next time it launches.

11 ## Building on Mac OS ##
12 VMD doesn't seem to support 64 bit builds on MacOS (at least no usable
13 pre-built ones are provided). By default, RMF is build 64 bit, and so
14 the plugin will not work. To build a 32 bit version with
15 [HomeBrew](http://brew.sh) do
16 - build `szip`, `hdf5`, `log4cxx` and `boost` with the `--universal` flag
17  (you have to uninstall them first).
18 - configure RMF with `-DCMAKE_OSX_ARCHITECTURES=i386`
19 - build it and link as above

22 # Caveats #
23 - If your version of VMD is not 1.9.1, pass the correct path for
24  finding the headers to cmake:
25  `-DVMDPLUGIN_INCLUDE_DIR=/path/to/vmd/plugin/include`. The directory
26  is the one that contains `molfile_plugin.h`.
27 - VMD's plugin model supports static geometry and atoms (or pseudo
28  atoms) that change with each frame. So no moving cylinders. Balls
29  are mapped to atoms.
30 - the plugin is linked against `libRMF` and `Avro` in the build directory
31 - it is not been tested on windows
