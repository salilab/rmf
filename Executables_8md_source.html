8 RMF provides a number of useful executables in the `bin` directory.
9 
10 # rmf_show # {#rmf_show}
11 `rmf_show` out the hierarchy in the file, optionally showing decorators and data.
12 
13  Show an rmf file as text.
14  Usage: ./bin/rmf_show rmf
15  -d [ --decorators ] Show what decorators recognize each node.
16  -f [ --frame ] arg Frame to use
17  -h [ --help ] Get help on command line arguments.
18  -v [ --verbose ] Show attribute values for each node.
19 
20 
21 # rmf_frames # {#rmf_frames}
22 `rmf_frames` prints out the frame hierarchy in the passed file.
23 
24  Print out information about frames
25  Usage: ./bin/rmf_frames rmf
26  -h [ --help ] Get help on command line arguments.
27 
28 # rmf_info # {#rmf_info}
29 
30 `rmf_info` displays information about the attributes found in the RMF and how
31 much they are used.
32 
33  Print out information about categories and keys.
34  Usage: ./bin/rmf_info rmf
35  -f [ --frame ] arg Frame to use
36  -h [ --help ] Get help on command line arguments.
37 
38 
39 # rmf_slice # {#rmf_slice}
40 
41 `rmf_slice` creates a new rmf from frames selected from an existing one.
42 
43  Grab frames from an rmf file
44  Usage: ./bin/rmf_slice input_rmf output_rmf
45  -f [ --frame ] arg First (or only) frame to use
46  -s [ --frame_step ] arg The step size for frames. Must be > 0.
47  -h [ --help ] Get help on command line arguments.
48 
49 # rmf_cat # {#rmf_cat}
50 
51 `rmf_cat` creates a new rmf by concatenating several. Each file must have the same hierarchical structure.
52 
53  Combine two or more rmf files.
54  Usage: ./bin/rmf_cat input_1.rmf input_2.rmf ... output.rmf
55  -h [ --help ] Get help on command line arguments.
56  -v [ --verbose ] Produce more output.
57  --hdf5-errors Show hdf5 errors.
58  --log-level arg What log level to use: Trace, Info, Warn, Error, Off
59 
60 
61 # rmf_xml # {#rmf_xml}
62 `rmf_xml` prints out the hierarchy and data to an XML files that can be opened in an XML viewer
63  (eg Google Chrome or Firefox). These viewers support collapsing of subtrees, which
64  makes it much easier to get around large hierarchies.
65 
66 
67 # rmf_validate # {#rmf_validate}
68 
69 `rmf_validate` checks for errors in an RMF file, it currently isn't very powerful.
70 
71 # rmf3_dump # {#rmf3_dump}
72 
73 `rmf3_dump` shows a low level dump of a 3rd generation RMF file (these are the current, avro-based files). It is primarily there for debugging.
